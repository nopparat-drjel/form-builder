# ADR-001: Monorepo with pnpm Workspaces

**Date:** 2025-08-01
**Status:** Accepted

## Decision
Use a pnpm workspaces monorepo (`apps/frontend`, `apps/worker`, `packages/shared`) instead of separate repositories.

## Rationale
Shared types and Zod schemas (`@hr-form/shared`) stay in sync automatically. A single `pnpm install` sets up the whole project. Atomic commits span frontend + backend changes.

## Consequences
Requires pnpm 9+. CI installs all packages before running any job. Simpler for a small team.
