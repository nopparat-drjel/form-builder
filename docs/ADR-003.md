# ADR-003: Cloudflare D1 as the Database

**Date:** 2025-08-01
**Status:** Accepted

## Decision
Use Cloudflare D1 (SQLite at the edge) rather than Postgres or PlanetScale.

## Rationale
D1 is co-located with Workers, so latency is minimal. SQLite's JSON support handles the `blocks` and `data` columns cleanly. Cost is zero until large scale. No connection pooling needed.

## Consequences
D1 is not globally replicated (single primary). For read-heavy future scale, add read replicas via the D1 API. Migrations use `wrangler d1 migrations apply`.
