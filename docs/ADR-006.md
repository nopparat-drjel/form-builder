# ADR-006: Form Block Storage as JSON in D1

**Date:** 2025-08-01
**Status:** Accepted

## Decision
Store the `blocks` array and response `data` as JSON strings in D1 TEXT columns rather than normalising blocks into a separate table.

## Rationale
Form schemas change rapidly during design iterations. JSON storage avoids costly schema migrations. D1's `json_extract` can query specific block fields if needed. Response data is fully schemaless by design.

## Consequences
Cannot efficiently query a specific field value across responses at the DB level without `json_each`. For analytics, export to R2/JSONL. Maximum D1 row size (1 MB) must be respected — forms with many blocks should stay well within this.
