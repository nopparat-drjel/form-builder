# ADR-002: Hono on Cloudflare Workers for the API

**Date:** 2025-08-01
**Status:** Accepted

## Decision
Build the backend as a Cloudflare Worker using the Hono framework instead of Express on a Node.js server.

## Rationale
Workers run at the edge with zero cold-start latency. Hono is typed end-to-end, lightweight (~14 kB), and supports D1/R2 bindings natively. No server to manage.

## Consequences
Must stay within Workers CPU/memory limits (50 ms CPU on free plan). Cannot use Node.js-only packages. bcryptjs is used instead of bcrypt (no native bindings).
