# ADR-004: JWT Authentication (Access + Refresh Tokens)

**Date:** 2025-08-01
**Status:** Accepted

## Decision
Issue short-lived access tokens (15 min) and long-lived refresh tokens (7 days) using `jose` and HS256.

## Rationale
Stateless access tokens work perfectly on Workers (no shared session store needed). Refresh tokens allow seamless UX without re-login. `jose` is a Web Crypto API implementation — no Node.js required.

## Consequences
Logout is client-side only unless a KV denylist is added. Token rotation happens transparently via the Axios interceptor in the frontend.
